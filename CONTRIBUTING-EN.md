# Requirements for Contributing

- Basic programming knowledge is required.

- Ability to use Git and GitHub.

- Some knowledge of Dart and how to use Flutter is preferable for an initial advantage.

- UI/UX Design skills are appreciated but not required.

- Some understanding of API requests or a willingness to learn.

# General Commitment Guidelines

If you want to add a feature or fix a bug, we'll be happy to accept pull requests! We only ask a few things:

- Ensure your code doesn't contain parser errors, for example:
- The code conforms to strong mode.

- The code is free of lint errors or warnings.

- Format the code with `Dart-Code.dart-code`.

- Write helpful documentation.

- If you would like to make a major/fundamental change to the codebase, please submit a level/issue PR.

# How to contribute

- Search for existing issues or create one in one of the projects. Then, create a branch/forward for any issue in a repository you are working on. Finally, commit your work.

- Create a Pull Request using the template, which will be simply detailed and will receive suggestions for improvements from the maintainers.

- Add screenshots or screenshots to your Pull Request to help me understand the effects of the proposed changes in your PR.

## Types of Contribution

- Bug fixes
- Code improvements
- Documentation
- Examples and tutorials
- Suggestions and ideas

## Best Practices

- Keep code clean and readable
- Ensure code is free of errors or lint warnings
- Follow project guidelines
- Use clear commit messages
- Be respectful in comments and reviews

## Pull Requests

- Use a clear and descriptive title
- Explain **what was done** and **why**
- Relate the pull request to an issue, if any

## Code of Conduct

By contributing, you agree to follow the community's [Code of Conduct](./CODE_OF_CONDUCT.md).
